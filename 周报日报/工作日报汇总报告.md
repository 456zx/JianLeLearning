## 三、主要问题与处理经验

### 1. 异步与热更环境问题

多次遇到 await、Forget、TryWait、out 参数、局部引用跨 await 丢失、ILRuntime Debug 模式时序错乱等问题。

处理经验：

- 关键异步流程尽量集中到一个函数中，用 await 明确顺序。
- TryWait 等等待结果需要判断返回值，失败时及时 return，避免退出游戏时卡死。
- 热更/ILRuntime 环境中尽量少依赖 out 参数跨 await 传值，可改为 return 对象或数组。
- 局部引用对象跨 await 后可能失效，重要字段应提前取出或减少跨异步作用域传递。
- Debug 和 Release 模式可能存在时序差异，需要分别验证。

### 2. UI 层级与点击问题

多次遇到按钮无法点击、点击穿透、空白关闭不生效、Canvas 层级混乱、UI 显示错位等问题。

处理经验：

- Button 需要 Image 或可射线检测组件配合，单独 Button 不一定能触发点击。
- 最外层页面可使用透明 Image + Button + Raycaster 实现空白点击关闭。
- UI 预制体根节点是否需要 Canvas/Raycaster，要结合统一 UI Layer 管理方式决定，避免多层 Canvas 导致层级混乱。
- 复用 UI 预制体时，需要注意根组件 Canvas 和 Raycaster，否则容易出现点击穿透或层级异常。
- 只修改场景中的 UI 不修改预制体，可能导致运行时不生效。

### 3. 打包与平台兼容问题

打包过程中遇到插件冲突、UnityEditor 引用、meta GUID 错误、包体过大、缓存未清、资源加载失败、平台 API 不支持等问题。

处理经验：

- 打包前检查本地服务器、缓存、DLL 更新、场景勾选、热更脚本挂载和资源路径。
- 非 Editor 文件夹下不能引用 UnityEditor.dll。
- meta 文件 GUID 异常可通过删除后让编辑器重新生成来修复。
- 平台不支持的 API 需要做条件编译或平台隔离。
- 微信小游戏首包限制可通过资源外置和动态加载方式缓解。

### 4. 配置表和资源管理问题

配置表字段名、类型、路径、资源名、占位符、颜色标记等都曾导致读取或显示问题。

处理经验：

- 表格字段名和代码字段名尽量保持一致，类型需要严格匹配。
- 对技能、勾玉、阵法等系统使用 enum 统一管理，减少字符串硬编码。
- 表格更新后要确保热更层读取到最新资源。
- 图片资源过大容易导致内存碎片和显示丢失，需要压缩或打包 SpriteAtlas。
- 资源路径和图标命名应与配置表规则保持一致。
